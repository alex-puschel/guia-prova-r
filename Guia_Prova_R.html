02 Gráfico de dispersão — dados da tabela manual ▶ 
# Criar o dataframe com os dados da tabela
horas <- c(5.2, 5.1, 4.9, 4.6, 4.7, 4.8, 4.6, 4.9)
tempo <- c(13, 15, 16, 20, 19, 17, 21, 16)
df_treino <- data.frame(horas = horas, tempo = tempo)

# Gráfico de dispersão simples
ggplot(df_treino, aes(x = horas, y = tempo)) +
  geom_point(size = 3, color = "steelblue") +
  labs(title = "Horas de Treinamento vs Tempo",
       x = "Horas de Treinamento", y = "Tempo para completar") +
  theme_minimal()

# Com reta de regressão
ggplot(df_treino, aes(x = horas, y = tempo)) +
  geom_point(size = 3, color = "steelblue") +
  geom_smooth(method = "lm", se = TRUE, color = "red") +
  labs(title = "Regressão Linear: Horas vs Tempo") +
  theme_minimal()
03 Ajustar e interpretar o modelo de regressão ▶ 
# Ajustar o modelo linear (y = tempo, x = horas)
modelo1 <- lm(tempo ~ horas, data = df_treino)
summary(modelo1)
coef(modelo1)   # mostra intercepto (β0) e inclinação (β1)
📖 Como ler o output do summary()
  • (Intercept) → β0: valor de y quando x = 0
  • horas (coeficiente) → β1: variação em y para cada +1 em x
  • R-squared → quanto o modelo explica da variação (0 a 1; quanto mais perto de 1, melhor)
  • p-value < 0.05 → relação estatisticamente significativa
  • Equação da reta: tempo = β0 + β1 × horas
# Gráfico com equação e correlação (pacote ggpubr)
ggscatter(df_treino, x = "horas", y = "tempo",
          add = "reg.line",
          add.params = list(color = "red"),
          conf.int = TRUE) +
  stat_cor(label.x = 4.8) +
  stat_regline_equation(label.x = 4.8, label.y = 21)
04 Análise / diagnóstico do modelo ▶ 
# 4 gráficos de diagnóstico de uma vez
par(mfrow = c(2,2))
plot(modelo1)
par(mfrow = c(1,1))  # resetar
📖 O que analisar nos gráficos de diagnóstico
  • Residuals vs Fitted: pontos aleatórios sem padrão = bom modelo
  • Q-Q Plot: pontos próximos da diagonal = resíduos normais = bom sinal
  • Scale-Location: homogeneidade da variância (linha horizontal = bom)
  • Cook's Distance: pontos influentes que podem distorcer o modelo

06 Regressão: horsepower → mpg ▶
💡 a) Intuição antes do código
  • Carros mais potentes tendem a consumir mais combustível
  • Logo, espera-se uma relação negativa: mais HP → menos MPG
# Dispersão primeiro
ggplot(auto, aes(x = horsepower, y = mpg)) +
  geom_point(alpha = 0.5, color = "steelblue") +
  labs(title = "Potência vs Eficiência (MPG)",
       x = "Potência (HP)", y = "Milhas por Galão") +
  theme_minimal()

# b) Ajustar o modelo
modelo_auto <- lm(mpg ~ horsepower, data = auto)
summary(modelo_auto)

# Gráfico com reta
ggplot(auto, aes(x = horsepower, y = mpg)) +
  geom_point(alpha = 0.5) +
  geom_smooth(method = "lm", color = "red", se = TRUE) +
  labs(title = "Regressão: Potência → MPG") +
  theme_minimal()
📊 Resultado esperado (b)
  • Coeficiente de horsepower será negativo (confirma intuição)
  • R² em torno de 0.60 → potência sozinha explica ~60% da variação do mpg
07 Apenas a potência é suficiente? Modelo múltiplo ▶ 
# Modelo com múltiplas variáveis
modelo_multi <- lm(mpg ~ horsepower + weight + cylinders + displacement,
                   data = auto)
summary(modelo_multi)

# Comparar R² dos dois modelos
cat("R² simples (só HP):", summary(modelo_auto)$r.squared, "\n")
cat("R² múltiplo:       ", summary(modelo_multi)$r.squared, "\n")
💬 Conclusão esperada
  • Apenas horsepower NÃO é suficiente — R² ~0.60 é moderado
  • O modelo múltiplo eleva R² para ~0.70+
  • Para boa previsão, precisamos de mais variáveis como weight e cylinders
